1. Motivating Example

Reaction time is the amount of time it takes a person to respond to a stimulus. In this project, I built a simple reaction time tester using an ESP32-S3 microcontroller, two pushbuttons, one LED, and one resistor.

The system works like a small reflex game:

  1. The user presses the Start Button.
  2. The ESP32-S3 waits for a random amount of time.
  3. The LED suddenly turns on.
  4. The user presses the Reaction Button as quickly as possible.
  5. The Serial Monitor displays the reaction time in milliseconds.
This project demonstrates digital input, digital output, timing with millis(), random delays, early-press detection, and Serial Monitor output.

For this tutorial, the project is implemented in Wokwi, an online electronics simulator that allows the ESP32-S3 circuit and Arduino-style C++ code to run virtually in a browser.

System Explanation

The project has two inputs and two outputs:

Category Component Purpose
Input Start Button Begins a new reaction-time test.
Input Reaction Button Records the user's response after the LED turns on.
Output LED Provides the visual signal to react to.
Output Serial Monitor Displays messages and the measured reaction time.

The ESP32-S3 controls the logic of the system. It waits for the start command, chooses a random delay, turns on the LED, measures the response time, and prints the final result.

4. Circuit Wiring Instructions

Pin Assignment

Function ESP32-S3 Pin
LED GPIO 4
Start Button GPIO 5
Reaction Button GPIO 6

Step 1: Connect the LED

  1. Connect GPIO 4 on the ESP32-S3 to one side of the resistor.
  2. Connect the other side of the resistor to the positive leg of the LED.
  3. Connect the negative leg of the LED to GND.
GPIO 4 → Resistor → LED → GND

Step 2: Connect the Start Button

  1. Connect one side of the Start Button to GPIO 5.
  2. Connect the opposite side of the Start Button to GND.
GPIO 5 → Start Button → GND

Step 3: Connect the Reaction Button

  1. Connect one side of the Reaction Button to GPIO 6.
  2. Connect the opposite side of the Reaction Button to GND.
GPIO 6 → Reaction Button → GND

Why the Buttons Connect to GND

The code uses the microcontroller's internal pull-up resistors:

pinMode(startButtonPin, INPUT_PULLUP);
pinMode(reactionButtonPin, INPUT_PULLUP);

This means:

  • When a button is not pressed, the pin reads HIGH.
  • When a button is pressed, the pin is connected to GND and reads LOW.

5. Source Code

// ESP32-S3 Reaction Time Tester
// LED: GPIO 4
// Start Button: GPIO 5
// Reaction Button: GPIO 6

const int ledPin = 4;
const int startButtonPin = 5;
const int reactionButtonPin = 6;

unsigned long lightOnTime = 0;
unsigned long reactionTime = 0;

void setup() {
  Serial.begin(115200);

  pinMode(ledPin, OUTPUT);

  // INPUT_PULLUP means:
  // not pressed = HIGH
  // pressed = LOW
  pinMode(startButtonPin, INPUT_PULLUP);
  pinMode(reactionButtonPin, INPUT_PULLUP);

  digitalWrite(ledPin, LOW);

  randomSeed(micros());

  Serial.println("=== Reaction Time Tester ===");
  Serial.println("Press the START button to begin.");
}

void loop() {
  // Wait until the Start Button is pressed
  if (digitalRead(startButtonPin) == LOW) {
    delay(200); // simple debounce

    Serial.println();
    Serial.println("Get ready...");
    digitalWrite(ledPin, LOW);

    // Random waiting time from 1 to 5 seconds
    int randomDelay = random(1000, 5001);

    unsigned long waitStart = millis();

    // During the waiting period, check if the player presses too early
    while (millis() - waitStart < randomDelay) {
      if (digitalRead(reactionButtonPin) == LOW) {
        Serial.println("Too early! You pressed before the light turned on.");
        Serial.println("Press START to try again.");

        // Wait until the player releases the button
        while (digitalRead(reactionButtonPin) == LOW) {
          delay(10);
        }

        return;
      }
    }

    // Turn on LED and start timing
    digitalWrite(ledPin, HIGH);
    lightOnTime = millis();

    Serial.println("GO! Press the REACTION button!");

    // Wait until Reaction Button is pressed
    while (digitalRead(reactionButtonPin) == HIGH) {
      // Keep waiting
    }

    reactionTime = millis() - lightOnTime;

    digitalWrite(ledPin, LOW);

    Serial.print("Your reaction time: ");
    Serial.print(reactionTime);
    Serial.println(" ms");

    Serial.println("Press START to play again.");

    // Wait until Reaction Button is released
    while (digitalRead(reactionButtonPin) == LOW) {
      delay(10);
    }

    delay(300);
  }
}

6. Most Interesting Parts of the Code

A. Internal Pull-Up Button Logic

pinMode(startButtonPin, INPUT_PULLUP);
pinMode(reactionButtonPin, INPUT_PULLUP);

The buttons are configured with internal pull-up resistors. The default reading is HIGH, and a pressed button reads LOW. Therefore, the code checks: 

if (digitalRead(startButtonPin) == LOW)

B. Random Waiting Time

int randomDelay = random(1000, 5001);

The ESP32-S3 chooses a random waiting period between 1 and 5 seconds. This prevents the player from predicting exactly when the LED will turn on.

C. Measuring Reaction Time

lightOnTime = millis();
reactionTime = millis() - lightOnTime;

The program records the moment the LED turns on. When the Reaction Button is pressed, it subtracts the stored start time from the current time to calculate the reaction time.

D. Detecting an Early Press

if (digitalRead(reactionButtonPin) == LOW)

During the random waiting period, the program checks whether the user presses too early. If so, it prints a warning message and asks the user to try again.

7. How the Program Runs on a Microcontroller

Although this version is demonstrated in Wokwi, the same type of code can be uploaded to a physical ESP32-S3 board.

  1. The programmer writes source code in Arduino-style C++.
  2. The development environment compiles the readable source code into machine-readable binary instructions.
  3. On physical hardware, the binary would be transferred to the ESP32-S3 through a USB cable.
  4. The microcontroller stores and executes those instructions.
  5. The processor repeatedly performs the instruction fetch, decode, and execute cycle to run the program.

In Wokwi, this behavior is simulated virtually, allowing the same circuit logic and program behavior to be demonstrated in a browser.
